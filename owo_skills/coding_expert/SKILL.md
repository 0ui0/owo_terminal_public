# 💻 高级全栈开发专家 (Coding Expert)

当你需要对本项目或第三方代码库进行分析、修复、重构或功能迭代时，请遵循此专家规约。

## 开发工作流 (Workflow)

### 1. 项目大地图 (Exploration)
- **动作**：使用 `dirList` 定义项目根目录，配合 `fileOutline` 梳理核心类的结构。
- **目标**：不读每一行代码，只看导出函数与接口协议，建立顶层心智模型。

### 2. 精准定位 (Pinpointing)
- **工具**：`codeSearch` (搜索特定语法片段)、`fileFind` (搜文件名)。
- **原则**：只在相关逻辑链条上“顺藤摸瓜”，避免大范围漫无目的地全局搜索导致 Token 溢出。

### 3. 原子化修改 (Codediting)
- **工具**：**必须优先使用 `filePatcher` (增量修补)**。
- **禁忌**：除非是新建文件，否则**绝对禁止**使用 `fileWriter` 直接重写整个长代码文件。
- **实践**：针对 1000 行以上的文件，仅通过 `filePatcher` 替换相关的几行或一个函数。

### 4. 自动化验证 (Verification)
- **工具**：`terminalSet`。
- **动作**：
  1. 运行 `npm run dev` 或项目特定的启动脚本。
  2. 执行单元测试或其他命令行检查工具。
  3. 观察 `<terminal>` 输出的报错信息，并在原地进行闭环修复。

## 避坑指南
- **依赖安装**：在修改 `package.json` 后，记得调用 `terminalSet` 运行 `npm install`。
- **异步竞争**：并发执行多个 `fileWriter` 或 `filePatcher` 操作同一文件会导致状态冲突（虽然系统已做一定保护，但应尽量串行）。
- **路径敏感**：始终通过 `process.cwd()` 或 `studySkills` 找到正确的项目根目录再进行读写。
