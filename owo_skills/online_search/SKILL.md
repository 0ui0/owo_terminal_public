# 🌐 全球信息搜寻专员 (Online Search Specialist)

当你需要从互联网获取实时信息、查找文档或分析在线内容时，请遵循此专家规约。

## 核心流程 (SOP)

### 1. 寻址与启动
- **动作**：使用 `browserLaunch` 启动浏览器并访问目标 URL（一般是搜索引擎如 Google/Baidu 或具体官网）。
- **注意**：系统已内置**自动唤醒机制**，即便浏览器当前处于最小化状态，该工具也会强制将其拉到前台完成操作。

### 2. 深度阅读 (Read-First)
- **动作**：优先调用 `browserGetContent` 获取页面的 Markdown 文本。
- **优势**：
  - **节省 Token**：Markdown 比原始 HTML 更加精简。
  - **快速扫描**：AI 可以迅速从中定位到关键标题和链接。

### 3. 多模式交互 (Interaction)
如果 Markdown 内容不足以解决问题（例如需要点击按钮、处理人机验证、或遇到单页应用渲染延迟）：
- **动作**：
  1. 使用 `browserScreenshot` 观测当前真实的 UI 布局。
  2. 使用 `browserInteract` 执行点击、输入或提交操作。
  3. 执行后配合 `waitMs` (等待 500ms~1000ms) 让渲染流水线完成。

### 4. 汇总与交付
- **动作**：从多个子页面提取信息，进行交叉验证。
- **输出**：提供带有来源说明的分类总结。

## 避坑指南
- **429 报错处理**：如果遇到频率限制，请适当增加 `waitMs` 延时。
- **多页抓取**：如果目标页面过长，`browserGetContent` 会自动截断。请查看底部页码提示，使用 `page` 参数翻页阅读。
- **最小化状态**：本系统工具会在操作结束后**自动恢复**浏览器的最小化状态，你无需手动点击最小化按钮。
