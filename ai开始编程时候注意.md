10. 代码规范：
  ①减少外部变量的使用，采用嵌入式写法，匿名或必要的时候命名
  命名情况主要用于会二次复用
  例如（语言无关，例子为coffeescript）：
  [].forEach (item)=>{}
  str = """
    #{
      output = do ->
        #一些逻辑
    }
  """.trim()
  不要这样写:
  callback = (item)=>{}
  [].forEach callback
  output = do-> 一些逻辑
  str = "#{output}"
  例外情况：除非嵌入的内容非常庞大和复杂，代码字符数量大于500
  这是为了能够线性理解和阅读代码，避免阅读的时候来回跳

  ②使用await必须配合使用try-catch，请确保最外层有try-catch
  try
    await fn()
  catch err
    console.log(err)
    throw err

  ③ 使用coffeescript，尽量省略括号、大括号，例如
  使用alert 1而不是 alert(1)
  使用
  console.log
    a:12
    b:5
  而不是
  console.log {
    a:12
    b:5
  }
  大括号只有使用展开运算符的时候需要提供
