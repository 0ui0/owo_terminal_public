# 项目计划：owoTimeMachine (时光机)

## 1. 项目目标
打造一个针对项目文件的“后悔药”系统。模仿 Mac 系统自带的 Time Machine，提供直观的线性时间轴、历史文件预览以及便捷的拖拽还原功能。

## 2. 核心技术逻辑
*   **备份引擎**：使用 Git CLI。
    *   **独立存储**：数据存储在项目根目录下的 `.owoTimeMachine/` 文件夹内。
    *   **身份隔离**：强制使用 `--git-dir` 和 `--work-tree` 参数，确保时光机的操作与用户项目原生的 `.git` 完全解耦。
*   **元数据管理**：
    *   **数据库**：使用 `.owoTimeMachine/snapshots.json`（轻量级 JSON 存储）。
    *   **线性逻辑**：放弃传统的 Git 分支逻辑，仅利用 Git 的快照存储能力。JSON 负责记录每一次备份的时间戳、用户描述及对应的 Git Commit Hash。
*   **还原机制**：
    *   **文件提取**：使用 `git show [hash]:[path]` 提取特定历史版本。
    *   **内部拖拽**：实现从“时光机窗口”到“资源管理器窗口”的 Drop 监听。放下鼠标即触发后端的文件提取与覆盖操作。

## 3. 功能规划
*   **初始化流程**：
    *   当项目首次加载或 App 启动时，主动询问用户是否开启时光机。
    *   引导用户完成第一个还原点的创建（Initial Snapshot）。
*   **时间轴导航**：
    *   垂直线性列表显示所有备份点。
    *   点击任意还原点，右侧视图切换为该时刻的“幽灵资源管理器”。
*   **幽灵资源管理器 (Ghost Explorer)**：
    *   复用 Explorer App 的 UI 组件，但数据源切换为 Git 历史对象。
    *   支持文件预览与拖拽发起。
*   **增量备份 (Snap)**：
    *   支持手动触发和（未来可能的）自动触发。
    *   自动处理忽略列表（排除 `.owoTimeMachine`, `.git`, `node_modules` 等）。

## 4. 文件结构
*   `server/apps/owoTimeMachine/`
    *   `backend.js`: 时光机逻辑处理（Git 调度、JSON 读写）。
    *   `frontend.js`: 时光机 UI（时间轴、幽灵资源管理器）。
    *   `项目计划.md`: 本文件。
*   `server/tools/timeMachineEngine.js`: 跨 App 调用的 Git 引擎封装。
