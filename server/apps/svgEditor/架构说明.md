绘图工具的逻辑是仿flash软件的
基本元件类：
线条
纯填充（纯填充内部会维护一个渲染用的边线数组，这个边线和画布元素无关，是内部自己用的）
填充组（维护逻辑关系，随时重建）
组 编组功能，所有操作只能反应到当前组及其递归的子组
  组内部持有对组内元素的引用，元素的真身在画布数组
  这个引用是乱序的，所以需要顺序必须从画布读取，它只是声明了一些元素属于组


基本工具：
墨水瓶 修改线条颜色，或者给纯填充创建填充组（会创建隐藏线条）
颜料桶 修改纯填充颜色，或者给闭合线条区域创建纯填充
  画布切割器逻辑会为其调用墨水瓶创建填充组

每次绘制的切割画布规则 splitBezier->main.coffee
1. 纯填充不能独立存在，会立即使用墨水瓶工具
2. active-false的隐藏线不能独立存在，会被清理
2. 近似的线条会被合并成一条，如果在填充组附近，会被合并进填充组
3. 线条和线条之间会相互切割
4. 填充组和线条之间会相互切割 先拆散组->切割->再填充
5. 填充组和填充组之间会相互切割 先拆散组->切割->再填充
6. 注意回形区域的处理

基本操作- 选择工具
选定 会根据层级和排序规则，组>组>元素
平移
  只有被选定的元素允许平移
  平移会拆散填充组，而后平移拆散前的组元素，平移后自动由画布切割规则重建关系（使用墨水瓶工具并合并线）
框选
  拖拽触发

元素层级说明：
组（1，2，3） > 普通元素
只有组能够排序
普通元素始终处于最底层，且相交会被切割
画布渲染前会重新排序给组置顶

正常数组元素添加是对纯填充(pureFill)使用unshift（置于数组最前/底层渲染），对线条及其他元素使用push（置于数组最后/顶层渲染）

渲染顺序和数组实际顺序不一定一致，需要阅读渲染代码

【工具类说明】
创建新工具的时候，需要到class目录注册工具类
class的结构需要参考已有的工具
工具需要到svgEditor_data.coffee使用init函数初始化
同时工具的对应事件需要在paperOncreate.coffee调用
同时工具对应的快捷，也在svgEditor_data.coffee的tools数组里维护
工具通常没有点击事件，由监听器检查当前工具来实现
此时工具可以配置完毕

通常工具的属性配置在工具类对应的类里
同时设置面板里的【工具属性】页面可以在前端修改工具属性

右键菜单在onPaperCreate里创建
