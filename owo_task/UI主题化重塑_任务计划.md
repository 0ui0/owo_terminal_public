# UI主题化重塑 - 任务计划书

## 1. 任务目标
通过重构 `getColor` 系统和适配核心 UI 组件，实现全站支持三套预设主题（欧式棕、明亮黄、经典蓝白），确保 UI 风格统一且易于扩展。

## 2. 任务清单

### 阶段 1：核心骨架重构
- [x] **重塑 `colorObj.js`**：建立纯净色表映射（Theme 0 基准），移除语义化 Key。
- [x] **重塑 `getColor.js`**：实现基于 `themeColor` 的三主题路由引擎。
- [x] **同步机制验证**：确保 `commonData.themeColor` 与后端配置同步。

### 阶段 2：外壳组件适配
- [x] **适配 `noticeBox.js`**：将所有硬编码 HEX 颜色替换为 `getColor` 调用。
- [x] **适配 `TitleBar.js`**：实现标题栏颜色动态化。
- [x] **适配 `nav.js`**：导航栏背景与边框适配。

### 阶段 3：内容组件适配
- [x] **适配 `ChatInputBar.js`**：输入框、按钮、附件区等 35+ 处硬编码替换。
- [x] **适配 `ChatItem.js`**：消息气泡、代码块等 40+ 处硬编码替换。
- [x] **适配 `ChatList.js`**：背景与边框适配。

### 阶段 4：全局样式与设置
- [x] **适配 `Css.js`**：滚动条及全局 HTML 背景使用模板字符串动态注入。
- [x] **适配 `setting.js`**：设置界面开关、按钮颜色适配。
- [x] **管理面板新增主题切换配置**：在设置界面提供 0/1/2 主题切换下拉/选择器。
- [x] **主题切换实时预览 (V17.1)**：点击即时生效，增强交互感。
- [x] **配色与名称优化 (V17.2)**：修正名称显示错误，黄/蓝主题采用“浅色背景+主题高亮”方案。
- [x] **深度色相适配与布局修正 (V17.3)**：实现色相感知的浅色化，修正全站背景硬编码。
- [x] **设计师级多彩重构 (V17.4)**：重塑为“阳光花园”与“蔚蓝海风”调色盘，实现图标与按钮的多彩化，采用纯白卡片设计。
- [x] **细节精修与透明度优化 (V17.5)**：实现实色气泡、半透明白色列表，修复输入文字可见性，同步终端动态配色。
- [x] **主题切换回归测试**：验证 0/1/2 三种模式下的显示效果及性能。

## 3. 验收标准
1. Theme 0 下显示效果需与重塑前 1:1 保持一致。
2. Theme 1 (明亮黄) 和 Theme 2 (经典蓝白) 显示正常，对比度符合可读性要求。
3. 切换主题后不刷新页面即可实时生效。
4. 全站无遗漏的硬编码 HEX 颜色值（排除 canvas 等特殊场景）。
